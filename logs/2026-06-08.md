## Bug Fixes

### 1. Tile-flip flash effect (`time.js`)

**Problem:** Every tile click triggered `renderBoard()`, which called `gameBoard.innerHTML = ''` and rebuilt all tile DOM elements from scratch. The brief empty-board state between clear and rebuild created a visible flash after each flip.

**Fix â€” three coordinated changes:**

**`flipTile()`** â€” stopped removing the `flipping` class after the animation completes. The tile stays visually flipped via the persistent class.
```js
// Before
setTimeout(() => {
    tileElement.classList.remove('flipping');
    this.isFlipping = false;
    resolve();
}, 300);

// After
setTimeout(() => {
    this.isFlipping = false;
    resolve();
}, 300);
```

**`renderBoard()`** â€” rewrote to update tiles in-place using `querySelector('[data-position]')` instead of clearing `innerHTML`. New tiles are only created on first render or if missing. The `flipping` class is only added if not already present (prevents re-triggering the CSS transition on already-flipped tiles).

**`resetTileAnimations()`** â€” added removal of both `resetting` and `flipping` when the reset animation completes, so tiles return cleanly to unflipped state between attempts.
```js
setTimeout(() => {
    element.classList.remove('resetting');
    element.classList.remove('flipping');  // added
    resolve();
}, 400);
```

### 2. Bottom counters resetting to 00 on refresh (`time.js`)

**Problem:** On page reload after completing the puzzle, `#liveTilesCount` and `#livePowersCount` both showed `00` because `flippedTiles` and `victoryStats` were empty on init.

**Fix:** In the completed-game loading path, restore `victoryStats` from the stored game result (which already contained `tilesRevealed` and `powersUsed`):
```js
if (gameResult.tilesRevealed !== undefined) {
    this.victoryStats.tilesRevealed = gameResult.tilesRevealed;
}
if (gameResult.powersUsed) {
    this.victoryStats.powersUsed = new Set(gameResult.powersUsed);
}
```

In `updateGameState()`, use `victoryStats.tilesRevealed` when the game is completed rather than the live `flippedTiles.size`:
```js
const tilesCount = this.isDailyCompleted
    ? this.victoryStats.tilesRevealed
    : this.flippedTiles.size;
if (tilesEl) tilesEl.textContent = String(tilesCount).padStart(2, '0');
```

### 3. Dark background peeking around flipped tiles (`game.css`)

**Problem:** After adding the tablet breakpoint (80â€“90px tiles), dark-panel color was visible at the rounded corners of flipped tiles. Caused by subpixel antialiasing gaps between the tile's background and the tile-face's clipped boundary â€” more visible at larger tile sizes with fractional-pixel `clamp()` + `vw` values.

**Fix:**
```css
/* Transparent bg on flipped tiles â€” gaps show cream instead of dark */
.tile.flipping {
    background-color: transparent;
}

/* Explicit inset: 0 instead of width/height: 100% to guarantee pixel-perfect coverage */
.tile-face {
    position: absolute;
    inset: 0;
    ...
}
```

## New Features

### 4. Completion state UI swap (HTML / `game.css` / `time.js`)

When the puzzle is completed, the three power-up icons and reset button are replaced by a full-width Stats button.

**HTML** â€” added `#statsBtn` to `.controls-row`:
```html
<button id="statsBtn" class="hidden">Stats</button>
```

**`game.css`** â€” Stats button spans the full controls row width:
```css
#statsBtn {
    flex: 1;
    height: clamp(42px, 10vw, 56px);
    font-size: clamp(1rem, 3.5vw, 1.3rem);
    ...
}
```

**`setupStatsButton()` in `time.js`** â€” hides `.power-icons` and `#restartBtn`, shows `#statsBtn`. `display: flex !important` on `#restartBtn` required using `style.setProperty('display', 'none', 'important')` to override specificity:
```js
setupStatsButton() {
    document.getElementById('statsBtn').classList.remove('hidden');
    document.querySelector('.power-icons').style.display = 'none';
    document.getElementById('restartBtn').style.setProperty('display', 'none', 'important');
}
```

Called from two paths: `showVictoryModal()` (winning now) and the completed-game load path in `initialize()`. The spurious call from `initialize()` that ran unconditionally on every page load was removed.

Stats button click wired to existing `showStatsModal()`.
