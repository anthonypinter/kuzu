## Bug Fixes

### 1. Dark stroke bleeding through flip cards (`tutorial.html`)

**Problem:** The flip cards on Page 3 were positioned as absolute overlays on top of the `page2-tiles.svg` row images. The underlying dark tile SVG (`#071b1e`) bled through at the card edges, showing as a dark stroke around the flower tiles. Attempts to fix with `overflow: hidden` / `overflow: visible` on `.flip-card-front` were insufficient.

**Root cause:** Two visual layers occupied the same space — the SVG img row (dark tile) and the flip card overlay. Subpixel alignment differences and the 3D perspective animation during the flip revealed the layer underneath.

**Fix:** Replaced the overlay approach entirely. When transitioning from Page 2 to Page 3, each `<img class="tut-tile-row">` SVG row is replaced with an `<div class="tut-tile-row-html">` containing 4 individual `<div class="tut-tile-cell">` elements. Flower positions get self-contained flip cards (`position: absolute; inset: 0` within their cell); non-flower positions get a static `<img>` of `page3-flowerback.svg`. No underlying dark tile layer exists, so nothing bleeds through.

```js
rows.forEach((rowEl, rowIdx) => {
    const newRow = document.createElement('div');
    newRow.className = 'tut-tile-row-html';
    for (let col = 0; col < 4; col++) {
        const cell = document.createElement('div');
        cell.className = 'tut-tile-cell';
        const flipIdx = flipMap.get(`${rowIdx}-${col}`);
        if (flipIdx !== undefined) {
            // flip card fills cell via position:absolute;inset:0
        } else {
            // static flowerback img fills cell
        }
        newRow.appendChild(cell);
    }
    tilesWrap.replaceChild(newRow, rowEl);
});
```

**CSS added:**
```css
.tut-tile-row-html {
    display: flex;
    align-items: center;
    padding: 0 calc(1.52 / 695.21 * 100%);
    gap: calc(24.47 / 695.21 * 100%);
    width: 100%;
    box-sizing: border-box;
    aspect-ratio: calc(695.21 / 157.74);
}
.tut-tile-row-html + .tut-tile-row-html { margin-top: 3.05%; }

.tut-tile-cell {
    flex: 1;
    aspect-ratio: 1;
    background: #071b1e;
    border-radius: 7.9%;
    position: relative;
}
```

The layout math preserves pixel-perfect parity with the SVG tile grid: left/right padding = `1.52/695.21`, inter-tile gap = `24.47/695.21`, each cell `flex: 1` → width = `154.69/695.21` of container.

## New Features

### 4. Page 4 — sequential title → text reveal, board stays, flowers jiggle

**Step definition:**
```js
{
    keepTiles:     true,
    sequenceText:  true,
    jiggleFlowers: true,
    title:         'tutorial/page4-title.svg',
    titleWidth:    '100%',
    text:          'tutorial/page4-text.svg',
    textWidth:     '95.8%',
}
```
Widths derived from viewBox widths relative to the 915.73px reference (`page4-title.svg` = 915.73 wide → 100%; `page4-text.svg` = 877.13 wide → 95.8%).

**Transition behaviour (Page 3 → Page 4):**
- Board stays in place (`keepTiles`)
- Old title and text fade out
- New title fades in with the header
- +400 ms: new text fades in (`sequenceText`)
- +300 ms after text starts: each flower tile jiggles, staggered 0.12 s apart (`jiggleFlowers`)

**Jiggle animation:**
```css
@keyframes tile-jiggle {
    0%   { transform: rotate(0deg); }
    20%  { transform: rotate(-6deg); }
    40%  { transform: rotate(6deg); }
    60%  { transform: rotate(-4deg); }
    80%  { transform: rotate(3deg); }
    100% { transform: rotate(0deg); }
}

.tut-tile-cell.jiggle {
    animation: tile-jiggle 0.65s ease-in-out both;
}
```

Applied to `.tut-tile-cell` (not `.flip-card-inner`) to avoid interfering with the 3D flip transform. A `void cell.offsetWidth` reflow forces the animation to restart cleanly on repeated visits to Page 4.

**JS trigger (inside keepTiles render branch):**
```js
if (steps[currentStep].sequenceText) {
    const newText = contentEl.querySelector('.tut-text');
    if (newText) {
        setTimeout(() => {
            newText.style.opacity = '';
            if (steps[currentStep].jiggleFlowers) {
                setTimeout(() => {
                    const cards = contentEl.querySelectorAll('.tut-tiles-wrap .flip-card');
                    cards.forEach((card, i) => {
                        const cell = card.parentElement;
                        cell.classList.remove('jiggle');
                        void cell.offsetWidth;
                        cell.style.animationDelay = (i * 0.12) + 's';
                        cell.classList.add('jiggle');
                    });
                }, 300);
            }
        }, 400);
    }
}
```

### 5. Backward navigation from Page 4 → Page 3

**Problem:** Page 4 is `keepTiles`, so backward nav to Page 3 keeps the illustration wrap. But if a future non-keepTiles page clears the DOM, Page 3's keepTiles code would find no `illusWrap` and render only text (no tile grid).

**Fix:** Added `buildRevealedTileGrid(step)` helper. If `illusWrap` is `null` and `step.flowers` is set, the grid is rebuilt from scratch with flowers already revealed (flip card inner set to `animation: none; transform: rotateY(180deg)`):

```js
} else if (step.flowers) {
    illusWrap = buildRevealedTileGrid(step);
    contentEl.appendChild(illusWrap);
}
```
