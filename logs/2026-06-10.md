### 2. Shared `window.__startGame()` function

Introduced to replace both the splash Enter button's handler and the tutorial play button's `window.location.href = 'index.html'` call. Detects which overlay is currently visible and fades it out before revealing the game.

```js
window.__startGame = function () {
    localStorage.setItem(STORAGE_KEY, todayStr);
    var active = (tutOverlay.style.display !== 'none') ? tutOverlay : overlay;
    active.style.transition = 'opacity 0.4s ease';
    active.style.opacity    = '0';
    active.addEventListener('transitionend', function () {
        overlay.style.display    = 'none';
        tutOverlay.style.display = 'none';
        gameWrapper.classList.remove('game-hidden');
        window.__splashActive = false;
        if (window.game) window.game.checkAndShowVictoryModal();
    }, { once: true });
};
```

### 3. "Learn How to Play" button routing

Added `id="splashHowToPlayBtn"` to the existing anchor. Click handler:

1. Adds `splash-out` class to fade the splash (0.6s)
2. On `transitionend`: hides splash, sets tutorial overlay `display: block; opacity: 0`

### 4. Background color scoped to overlay (Page 5)

Page 5's dark-green (#1b4042) background transition previously targeted `document.documentElement` and `document.body`. All three occurrences (forward to page 5, backward from page 5, forward to page 6) were updated to target `tutOverlayEl` instead, so the color change is scoped to the overlay and doesn't affect the rest of the page.

```js
// Before
document.documentElement.style.backgroundColor = '#1b4042';
document.body.style.backgroundColor            = '#1b4042';

// After
tutOverlayEl.style.backgroundColor = '#1b4042';
```

### 5. Page 7 — Left-align tiles after fade

After unflipped tiles fade out, remaining flipped tiles slide left to align with column 0 using `getBoundingClientRect()` for pixel-accurate `translateX`:

```js
contentEl.querySelectorAll('.tut-tile-row-html').forEach(rowEl => {
    const cells = rowEl.querySelectorAll('.tut-tile-cell');
    if (!cells.length) return;
    const targetLeft = cells[0].getBoundingClientRect().left;
    cells.forEach(cell => {
        if (!cell.querySelector('.flip-card')) return;
        const dx = targetLeft - cell.getBoundingClientRect().left;
        if (dx !== 0) {
            cell.style.transition = 'transform 0.4s ease';
            cell.style.transform  = `translateX(${dx}px)`;
        }
    });
});
```

### 6. Page 7 — Power label SVGs (uniform size)

Five SVG labels (potion, shears, flower, portal, grapple) fade in next to their respective tile icons. Sizing problem: all SVGs have the same viewBox height (96.5) but different widths (391–542). With `object-fit: contain`, each SVG scaled to its own aspect ratio, producing different text heights.

**Fix:** Compute `maxAspect` from the widest SVG, then derive `labelBoxH = sharedWidth / maxAspect`. This makes the widest SVG exactly fill the available width, and all others become height-constrained at the same rendered height.

```js
const maxAspect = Math.max(...step.powerLabels.map(p => p.svgW / p.svgH));
const labelBoxH = sharedWidth / maxAspect;
```

Each label is an `<img>` with `object-fit: contain; object-position: left center` positioned absolutely within the illustration wrap.

SVG dimensions (from viewBox):
| Label | Width | Height |
|-------|-------|--------|
| page7-potion.svg  | 486.23 | 96.5 |
| page7-shears.svg  | 542.77 | 96.5 |
| page7-flower.svg  | 542.39 | 96.5 |
| page7-portal.svg  | 479.32 | 96.5 |
| page7-grapple.svg | 391.88 | 96.5 |

## Architecture Notes

- Tutorial JS is wrapped in an IIFE to avoid polluting global scope; exposes only `window.renderTutorial`
- `render(false)` is NOT called at script parse time — called by the splash handler once the overlay is visible
- `tutorial.html` is retained as a standalone development reference but is no longer linked from the main app
